# Integrate APNs


For iOS devices to implement the offline push notification feature by ZIM SDK, it uses the Apple Push Notification service (APNs) and will need you to follow the steps below to apply for a certificate on [Apple Developer](https://developer.apple.com/) first.



## Implementation steps

1. Sign in to [Apple Developer](https://developer.apple.com/) and click **Account** in the top navigation bar.
<Frame width="512" height="auto" caption="">
  <img src="https://storage.zego.im/sdk-doc/Pics/ZIM/create_apns_account.png" />
</Frame>

2. On the page that appears, click **Certificates, IDs & Profiles**.
<Frame width="512" height="auto" caption="">
  <img src="https://storage.zego.im/sdk-doc/Pics/ZIM/certificates_ids.png" />
</Frame>

3. Click the plus icon "+" next to **Identifiers** and enter the App ID **Name** and **Bundle ID**.

    <Note title="Note">
    - Skip this step if the Apple ID already exists.
    - Bundle ID cannot contain any wildcard characters.
    </Note>

<Frame width="512" height="auto" caption="">
  <img src="https://storage.zego.im/sdk-doc/Pics/ZIM/apns_apply_3.png" />
</Frame>

4. Check **Push Notification**.

<Frame width="512" height="auto" caption="">
  <img src="https://storage.zego.im/sdk-doc/Pics/ZIM/select_push_notification.png" />
</Frame>


5. Apply for a .p12 certificate.

    In the **Certificates, IDs & Profiles** section, click the plus icon "+" next to **Certificates**.

<Frame width="512" height="auto" caption="">
  <img src="https://storage.zego.im/sdk-doc/Pics/ZIM/apns_apply_2.png" />
</Frame>
<Frame width="512" height="auto" caption="">
  <img src="https://storage.zego.im/sdk-doc/Pics/ZIM/create_new_certificates_EN.png" />
</Frame>

    Select **Apple Push Notification service SSL (Sandbox & Production)** and an App ID for binding to the certificate. Then, click **Continue**.
    
<Frame width="512" height="auto" caption="">
  <img src="https://storage.zego.im/sdk-doc/Pics/ZIM/apns_apply_4.png" />
</Frame>

    Select and upload a Certificate Signing Request (CSR) file and click **Continue**. For information on how to obtain a CSR file, see Steps 6 and 7.

<Frame width="512" height="auto" caption="">
  <img src="https://storage.zego.im/sdk-doc/Pics/ZIM/upload_csr_file.png" />
</Frame>

6. Open Keychain Access on your iOS device.

<Frame width="512" height="auto" caption="">
  <img src="https://storage.zego.im/sdk-doc/Pics/ZIM/open_keychain_access_EN.png" />
</Frame>

7. Choose **Keychain Access** > **Certificate Assistant** > **Request a Certificate from a Certificate Authority**. Enter the user's email address and a common name. Select **Saved to disk** and click **Continue** to get a .certSigningRequest file (CSR file).

<Frame width="512" height="auto" caption="">
  <img src="https://storage.zego.im/sdk-doc/Pics/ZIM/request_certificate_EN.png" />
</Frame>
<Frame width="512" height="auto" caption="">
  <img src="https://storage.zego.im/sdk-doc/Pics/ZIM/apns_apply_1_EN.png" />
</Frame>

8. Upload the obtained `.certSigningRequest` file on the page stated in Step 5 and click **Continue**. Open the download page and click **Download** to get a `.cer` file. Double-click this file to install it properly in Keychain Access. As shown in the figure below, locate the certificate file and export a `.p12` file.
<Frame width="512" height="auto" caption="">
  <img src="https://storage.zego.im/sdk-doc/Pics/ZIM/export_certificate_EN.png" />
</Frame>

9. Upload the iOS certificate file:

- Log in to the [ZEGOCLOUD Admin Console](https://console.zegocloud.com/), select **Service Management > In-app Chat** tab, and add APN's certificates in the **Notification configuration** session. (Select the corresponding configuration when you upload the certificate.)
<Frame width="512" height="auto" caption="">
  <img src="https://storage.zego.im/sdk-doc/Pics/Console_APNs_01.png" />
</Frame>
<Frame width="512" height="auto" caption="">
  <img src="https://storage.zego.im/sdk-doc/Pics/Console_APNs_02.png" />
</Frame>

    At this point, you have completed the configuration of the APNs offline push notification.
