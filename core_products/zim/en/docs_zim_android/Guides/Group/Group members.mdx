# Manage group members

- - -

ZEGOCLOUD's In-app Chat (the ZIM SDK) provides the capability of group member management, allowing you to query the group member list and group member info, change the group owner, and more.



<Warning title="Warning">

- Before you use the following features, you must join a group first. To join a group, refer to the Chapter **Join a group** of [Manage groups](./Manage%20groups.mdx).
</Warning>

<Frame width="512" height="auto" caption=""><img src="https://storage.zego.im/sdk-doc/Pics/in_app_chat/14556.png" /></Frame>

## Get group member list


To know who is in the current group and get the group member list after login, call the {getPlatformData(props,queryGroupMemberListMap)} method. A single call to the interface can obtain up to 100 members. If the passed `count` exceeds 100, it will be processed as 100.

You will receive a notification through the callback {getPlatformData(props,ZIMGroupMemberInfoQueriedCallbackMap)}  after the query succeeds. 

<Note title="Note">

As of the current version, when this interface is called for the first time to pull the list of group members, the avatars of group members can be retrieved. If you call this interface again, you can get the avatar information of new users, but the avatar information of existing users will not be updated.

To get the latest avatars of group members, please call the interface {getPlatformData(props,queryGroupMemberInfoMap)}  <a href="#queryGroupMemberInfo">to query group member info</a>.
</Note>

## Set a role for a group member


For a group owner to set a role for a group member, call the {getPlatformData(props,setGroupMemberRoleMap)} method. You can set the group member to a regular member or admin.

After the setting is successful, the group owner can receive the setting result through {getPlatformData(props,ZIMGroupMemberRoleUpdatedCallbackMap)} .

Group members can receive notifications of user role changes through {getPlatformData(props,onGroupMemberInfoUpdatedMap)}.
   



#### Description of group member roles and permissions
The ZIM SDK supports setting users as group owners, administrators, and regular members by default. In a group, the group owner has all **client permissions** and can perform all group functions. Administrators have most **client permissions**, while regular members have the fewest **client permissions**. The specific client permissions for each role are shown in the table below:

## Transfer the group ownership


For a group owner to transfer the group ownership, call the {getPlatformData(props,transferGroupOwnerMap)} method with the `toUserID` (the ID of the member you want to transfer the group ownership to).

<Warning title="Warning">

- The member you want to transfer the group ownership to must be in the group; otherwise, the operation fails.
</Warning>

All group members will receive a notification through the callback {getPlatformData(props,onGroupMemberInfoUpdatedMap)}  after the group owner is changed.

### Search group members

After logging in to the ZIM SDK and joining a group, if you want to search for users within the group based on certain criteria, use the  {getPlatformData(props,searchLocalGroupMembersMap)} interface. By providing the groupID、config、callback, you can search for group members that meet the specified conditions.

The search results will be returned through the {getPlatformData(props,ZIMGroupMembersSearchedCallbackMap)} interface.




:::if{props.platform=undefined}
<CodeGroup>
```java
// Search a group for members with "zego" in their name
ZIM zim = getZIM();
ZIMGroupMemberSearchConfig config = new ZIMGroupMemberSearchConfig();
config.count = 10;
config.nextFlag = nextFlag;
config.isAlsoMatchGroupMemberNickname = true; // If a group member's nickname contains zego, the search results will include that member
config.keywords.add("zego");
zim.searchLocalGroupMembers(conversationID, config, new ZIMGroupMembersSearchedCallback() {
    @Override
    public void onGroupMembersSearched(String groupID, ArrayList<ZIMGroupMemberInfo> userList, int nextFlag, ZIMError errorInfo) {
        // Developers can obtain group member information from userList
```

### Set Mute status for group members
After logging into the ZIM SDK, you can mute or unmute specific members within the groups they manage. By calling the muteGroupMembers|_blank interface, you can modify the mute status of up to 100 group members in bulk. The mute duration can be permanent or up to a maximum of 604,800 seconds (7 days).

<Note title="Note">

- The group owner can prohibit all group members, including themselves, from speaking.
- If you need to increase the number of operations per request or extend the maximum mute duration, please contact ZEGOCLOUD technical support team.
</Note>

After the settings are successfully applied, the operating user will receive notifications through {getPlatformData(props,ZIMGroupMembersMutedCallbackMap)} .

Once a mute or unmute operation is successful, all group members will receive {getPlatformData(props,onGroupMemberInfoUpdatedMap)}, which informs them about the members who are unable to speak in the group or have had their mute status lifted.

<Note title="Note">

If you want to mute certain group members , please refer [Manage groups - Mute group](./Manage%20groups.mdx#mute-or-unmute-a-group).
</Note>

### Get the mute status of current group members

To actively retrieve the mute status of the current user in a group, please use any of the following methods:
- Refer to [Manage sessions - Get the session list](./../Conversation/Get%20the%20conversation%20list.mdx) to fetch the session list proactively.
- Refer to [Manage sessions - Update session list](./../Conversation/Get%20the%20conversation%20list.mdx) for session change callbacks and update the session list.

Retrieve the {getPlatformData(props,mutedExpiredTimeMap)} from the {getPlatformData(props,ZIMConversationMap)} object in the returned result, which indicates the mute duration in seconds.

The explanation of the {getPlatformData(props,mutedExpiredTimeMap)} value is as follows:
- When it is -1, it means the current user cannot speak in that group permanently.
- When it is 0, it means the current user can speak in that group.
- When it is 0, it means the current user can speak in that group.

<Note title="Note">


If the current user is both muted due to group role restrictions (refer to [Manage groups - Mute or unmute group](./Manage%20groups.mdx), and individually muted（refer to the section <a href="#set-mute-status-for-group-members">Set mute status for group members</a> in this article）, the value of {getPlatformData(props,mutedExpiredTimeMap)} will be determined by taking the maximum value between the "group mute time" and the "individual mute time".

</Note>
