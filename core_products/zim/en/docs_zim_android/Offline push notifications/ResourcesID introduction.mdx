# ResourcesID introduction


ResourcesID is an string that can map advanced push configurations for multiple push service providers. It has a maximum length of 32 characters and can be customized by you. By using this field, you can abstract the push configurations of multiple vendors into a custom push strategy.

By default, a project can have up to 10 resourcesIDs at the same time. If you need to increase this limit, please contact ZEGOCLOUD technical support.

## Configuration

1. Log in to the ZEGOCLOUD console, and naviagte to the "Projects > Projects Management" tab, select one project to enter the "Project Configuration" page.

2. Select the "Service Management > In-app Chat" tab.

    <Frame width="512" height="auto" caption=""><img src="https://storage.zego.im/sdk-doc/Pics/ZEGOCLOUD_Console/service_management_1.jpeg" /></Frame>

3. Find the "Customized push resource" and click "Add".

    <Frame width="512" height="auto" caption=""><img src="https://storage.zego.im/sdk-doc/Pics/ZEGOCLOUD_Console/service_management_2.jpeg" /></Frame>

4. Enter the relevant information for the resourcesID in the pop-up window and click "Continue".

5. After completing the input, the specific information of the resourcesID will appear under the "Customized push resource".

    <Frame width="512" height="auto" caption=""><img src="https://storage.zego.im/sdk-doc/Pics/ZIM/OfflinePush/Copy_1.jpeg" /></Frame>

## Usage

You need to pass the resourcesID field when calling ZIM interfaces that include the `pushConfig` parameter. The ZPNs server will select the appropriate push service provider configuration based on the receiver's push service provider to complete the push.
